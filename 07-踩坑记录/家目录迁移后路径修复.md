# 家目录迁移后路径修复

## 问题背景
重装系统后，家目录从 `/home/junjun` 改为 `/home/jun`，导致残留旧路径引发权限错误。

## 错误表现

```
EACCES: permission denied, mkdir '/home/junjun'
```

定时任务执行时报错，因为会话状态文件中残留旧路径。

## 修复步骤

### 1. 检查残留路径

```bash
# 查找包含旧路径的配置文件
grep -r "/home/junjun" ~/.openclaw/ 2>/dev/null
```

### 2. 清理会话状态

```bash
# 批量替换旧路径
find ~/.openclaw -name "*.json" -exec sed -i 's|/home/junjun|/home/jun|g' {} \;
```

### 3. 重点检查文件

- `~/.openclaw/agents/*/sessions/sessions.json`
- `~/.openclaw/gateway/openclaw.json`
- `~/.openclaw/*/memory/*.md`

### 4. 验证修复

```bash
# 确认无残留
grep -r "/home/junjun" ~/.openclaw/ 2>/dev/null

# 重启 gateway
openclaw gateway restart
```

## 预防措施

1. 重装系统前备份配置
2. 使用 `$HOME` 而非绝对路径
3. 配置文件使用相对路径
