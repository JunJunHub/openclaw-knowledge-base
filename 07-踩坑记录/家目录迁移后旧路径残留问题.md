# 家目录迁移后旧路径残留问题

## 问题现象

cron 任务执行时报错：
```
EACCES: permission denied, mkdir '/home/junjun'
```

用户已重装系统，家目录从 `/home/junjun` 改为 `/home/jun`，但旧路径仍存在于配置中。

## 问题原因

会话状态文件中残留旧路径：
- `logs/config-health.json` - 日志
- `agents/*/sessions/sessions.json` - 会话状态

这些历史记录导致 message 工具尝试访问旧路径。

## 解决方案

```bash
# 清理所有 JSON 文件中的旧路径
find ~/.openclaw -name "*.json" -type f -exec sed -i 's|/home/junjun|/home/jun|g' {} \;
```

**需要重启 Gateway 才能生效**：
```bash
openclaw gateway restart
```

## 验证

```bash
# 检查是否还有旧路径
grep -r "/home/junjun" ~/.openclaw/*.json 2>/dev/null
```
