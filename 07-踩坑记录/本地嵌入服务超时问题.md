## 问题描述

在配置 Memory Search 时，使用 `node-llama-cpp` 本地嵌入服务出现超时问题。

## 问题原因

1. **本地嵌入模型性能不足**：虚拟机环境无独立 GPU，导致嵌入计算超时
2. **模型 token 限制**：`BAAI/bge-large-zh-v1.5` 只支持 512 tokens，无法处理长文本

## 解决方案

### 1. 改用远程嵌入服务

将 Memory Search 的嵌入服务从本地改为 SiliconFlow 远程服务：

```json
{
  "memorySearch": {
    "provider": "openai",
    "model": "BAAI/bge-m3",
    "mode": "hybrid"
  }
}
```

### 2. 使用支持长文本的模型

- **旧模型**: `BAAI/bge-large-zh-v1.5`（512 tokens）
- **新模型**: `BAAI/bge-m3`（8192 tokens）

## 配置说明

### SiliconFlow 嵌入服务配置

1. 确保 `SILICONFLOW_API_KEY` 已配置
2. 在 `openclaw.json` 中配置：

```json
{
  "memorySearch": {
    "provider": "openai",
    "model": "BAAI/bge-m3",
    "mode": "hybrid",
    "openai": {
      "baseURL": "https://api.siliconflow.cn/v1",
      "apiKey": "${SILICONFLOW_API_KEY}"
    }
  }
}
```
