# 问题

定时任务（Cron Job）执行后，飞书消息发送失败：

```
⚠️ 飞书发送失败
飞书频道不可用，需安装 `@openclaw/feishu` 插件
```

# 原因分析

定时任务的 Message Delivery 机制与实时聊天信道不同：

- **实时聊天**：通过 `channels.feishu` 内置信道处理
- **定时任务发送**：需要额外的 Message 插件支持

# 解决方案

## 方案一：安装飞书插件

```bash
openclaw plugins install @openclaw/feishu
```

## 方案二：使用其他发送渠道

在 Cron Job 配置中修改 `delivery` 配置：

```json
{
  "delivery": {
    "mode": "announce",
    "channel": "telegram",
    "to": "your_chat_id"
  }
}
```

## 方案三：使用 Webhook

```json
{
  "delivery": {
    "mode": "webhook",
    "to": "https://open.feishu.cn/open-apis/bot/v2/hook/xxx"
  }
}
```

# 注意事项

⚠️ 如果同时启用了飞书信道和飞书插件，可能产生路径冲突。参考 [[飞书插件路径冲突问题]]。
