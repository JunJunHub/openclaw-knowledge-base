# 大模型上下文窗口查看方法

## 背景
用户询问如何知道大模型的上下文窗口大小，以百度千帆 GLM-5.0 为例。

## 三种查看方法

### 方法 1：查官方文档/API 规格
每个模型发布时都会标注：
- GPT-4 Turbo: 128K tokens
- Claude 3.5 Sonnet: 200K tokens
- Qwen2.5: 32K~1M（按版本不同）
- 百度千帆文心系列：32K（部分型号）

> **注意**：`qianfan-code-latest` 是千帆的"动态模型"，会自动切换到最新版本，窗口大小可能随时间变化。

### 方法 2：问 API 返回信息
有些 API 会在响应头或模型信息接口返回 `max_tokens`：

```bash
# OpenAI 兼容接口
curl https://api.example.com/v1/models -H "Authorization: Bearer $KEY"
```

### 方法 3：实测（最靠谱）
发送超长 prompt，观察什么时候截断或报错：

```python
# 测试脚本
for size in [4000, 8000, 16000, 32000, 64000]:
    text = "x" * size
    try:
        response = model.chat(text)
        print(f"{size} tokens: OK")
    except:
        print(f"{size} tokens: FAILED")
```

## GLM-5.0 规格

| 参数 | 值 |
|------|-----|
| **上下文窗口** | **200K tokens** |
| **最大输出** | 128K tokens |
| **参数量** | 744B（激活 40B）|
| **发布时间** | 2026年2月（智谱AI）|

训练时分三阶段扩展：32K → 128K → 200K，专门针对长文档和代码库优化。

### 200K 是什么概念？
- ≈ 15 万汉字
- ≈ 一本中篇小说
- ≈ 整个小型代码库
