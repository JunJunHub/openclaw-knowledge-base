# 小红书笔记去AI化处理指南

## 背景
小红书平台对 AI 生成内容有检测机制，直接发布 AI 文案可能导致封号。

## AI 文案的特征痕迹

| 痕迹类型 | 示例 |
|---------|------|
| 格式太整齐 | 表格、分隔线、进度条 |
| emoji 过于规律 | 每段都有 ✨ 💡 📌 |
| 模板化结构 | 坑1/坑2/坑3、第一/第二/第三 |
| 空洞赞美 | "这个功能太实用了 ✨" |
| 标签过多 | 5-7 个标签 |

## 去AI化改写原则

1. **口语化表达**：用日常对话语气
2. **适度用颜文字**：1-2 个/篇，不规律
3. **句式参差**：长短句交替，避免对称结构
4. **加入真实细节**：具体数字、真实场景
5. **标题动词开头**：如"做了个..."、"发现..."、"踩了个坑..."

## 处理流程

```
原 AI 文案 → 拆分系列 → 逐篇改写 → 控制字数(300-500) → 标签精简(2-3个)
```

## 示例对比

**❌ AI 风格：**
```
## 三大踩坑记录

坑1：配置文件路径错误 ❌
解决方法：修改 config.yaml...

坑2：依赖版本冲突 ⚠️
解决方法：使用 pnpm...

坑3：权限不足 🔒
解决方法：sudo chmod...
```

**✅ 去AI化：**
```
昨天踩了个坑，配了一下午环境。

改 config.yaml 的时候路径写错了，找了半天才发现。后来依赖又冲突，换了 pnpm 才行。最后还要 sudo 改权限...

总结：装东西前先看文档，能省不少时间。
```
