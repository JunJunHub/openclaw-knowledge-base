# 家目录迁移后路径修复

> **分类**：安装部署
> **相关文档**：[[openclaw-recovery一键恢复工具]]

## 背景

重装系统后家目录变更：
- 旧路径：`/home/junjun`
- 新路径：`/home/jun`

## 需要检查的配置文件

| 文件 | 位置 | 检查内容 |
|------|------|---------|
| `openclaw.json` | `~/.openclaw/` | 工作空间路径、浏览器路径 |
| `exec-approvals.json` | `~/.openclaw/` | socket 路径 |
| `MEMORY.md` | `~/.openclaw/workspace/` | 家目录引用 |
| `.bashrc` | `~/` | 环境变量路径 |
| SSH 配置 | `~/.ssh/` | 密钥文件存在 |

## 批量替换命令

```bash
# 查找旧路径
grep -r "/home/junjun" ~/.openclaw/ --include="*.json" --include="*.md"

# 批量替换（谨慎使用）
find ~/.openclaw/ -type f \( -name "*.json" -o -name "*.md" \) -exec sed -i 's|/home/junjun|/home/jun|g' {} \;
```

## 环境变量检查

```bash
echo $HOME
echo $PATH
```

## 历史会话记录

Sessions 文件中可能有大量旧路径引用，这些是历史日志，不影响运行，可以忽略。
